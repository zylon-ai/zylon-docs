Usa los endpoints de cuentas de Backoffice para gestionar identidades de plataforma y tokens de API asociados a cuenta.

## Requisitos previos

- Token con permisos de backoffice en `Authorization: Bearer {API_TOKEN}`
- URL base objetivo (`https://{BASE_URL}`)

Antes de asignar o actualizar roles, revisa [Roles y permisos](/es/user-manual/backoffice/roles-and-permissions).

## Cuentas

### Crear una cuenta

Usa `POST /api/v1/auth/register` para crear una cuenta con credenciales.

```bash
curl -X POST "https://{BASE_URL}/api/v1/auth/register" \
  -H "Authorization: Bearer {API_TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{
    "email": "{username}",
    "password": "{password}",
    "roles": ["AppAdmin"]
  }'
```

## Tokens

Los tokens de cuenta est獺n pensados para acceso a PrivateGPT API y Workspace API.

- `gateway_id` es obligatorio al crear tokens de cuenta.
- Para crear tokens de uso exclusivo en Workspace, usa [Gateways - Crear token de gateway](/es/developer-manual/build-with-zylon/backoffice-api/gateways#crear-token-de-gateway).

### Crear token API de cuenta

```bash
curl -X POST "https://{BASE_URL}/api/v1/account/token" \
  -H "Authorization: Bearer {API_TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "workspace-automation",
    "gateway_id": "gw_9a2e9f4f",
    "valid_until": "2027-12-31T00:00:00Z",
    "revoke_existing": false
  }'
```

## Errores comunes

- Mezclar token en query y token en path en `/v1/account/token/{token?}`.
- Intentar actualizar roles de cuenta con tokens sin permisos de backoffice.
- Omitir `Content-Type: application/json` en endpoints con body.
