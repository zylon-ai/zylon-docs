Usa los endpoints de cuentas de Backoffice para gestionar identidades de plataforma y tokens de API asociados a cuenta.

## Requisitos previos

- Token con permisos de backoffice en `Authorization: Bearer {API_TOKEN}`
- URL base objetivo (`https://{BASE_URL}`)

Antes de asignar o actualizar roles, revisa [Roles y permisos](/es/user-manual/backoffice/roles-and-permissions).

## Cuentas

### Roles de cuenta disponibles

Los roles de cuenta controlan el acceso a nivel de plataforma:

- `Workspace`: Usa Workspace para proyectos y colaboración. No tiene acceso a Platform Backoffice.
- `Developer`: Trabaja con gateways y tokens API para desarrollo. Acceso a Developer Tools.
- `Operator`: Gestiona cuentas, roles, gateways y operaciones de plataforma. Acceso completo a Platform Backoffice.
- `SuperAdmin`: Cuenta administrativa inicial para configuración y gobierno de plataforma. Mismo acceso de plataforma que `Operator`.

### Crear una cuenta

Usa `POST /api/v1/auth/register` para crear una cuenta con credenciales.

```bash
curl -X POST "https://{BASE_URL}/api/v1/auth/register" \
  -H "Authorization: Bearer {API_TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{
    "email": "{username}",
    "password": "{password}",
    "roles": ["Workspace"]
  }'
```

### Actualizar contraseña de la cuenta actual

```bash
curl -X PUT "https://{BASE_URL}/api/v1/account/me" \
  -H "Authorization: Bearer {API_TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{
    "current_password": "{current_password}",
    "new_password": "{new_password}"
  }'
```

<Accordion title="Ejemplo de respuesta (400 si la contraseña actual es inválida)">
```json
{
  "type": "InvalidPassword",
  "status": 400,
  "message": "Invalid current password"
}
```
</Accordion>

## Tokens

Los tokens de cuenta están pensados para acceso a PrivateGPT API y Workspace API.

- `gateway_id` es obligatorio al crear tokens de cuenta.
- Para crear tokens de uso exclusivo en Workspace, usa [Gateways - Crear un token de gateway](/es/developer-manual/build-with-zylon/backoffice-api/gateways#crear-un-token-de-gateway).

### Crear token API de cuenta

```bash
curl -X POST "https://{BASE_URL}/api/v1/account/token" \
  -H "Authorization: Bearer {API_TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "docs-account-final-1772554276",
    "gateway_id": "{gatewayId}",
    "scopes": ["Workspace.ReadWrite.All"],
    "valid_until": "2027-12-31T00:00:00Z",
    "revoke_existing": false
  }'
```

### Eliminar token API de cuenta

```bash
curl -X DELETE "https://{BASE_URL}/api/v1/account/token/{token_hash}" \
  -H "Authorization: Bearer {API_TOKEN}"
```

<Accordion title="Ejemplo de respuesta">
```text
Token deleted
```
</Accordion>

## Errores comunes

- Mezclar token en query y token en path al trabajar con `/api/v1/account/token/{token_hash}`.
- Intentar actualizar roles de cuenta con tokens sin permisos de backoffice.
- Omitir `Content-Type: application/json` en endpoints con body.

## Errores y casos límite

- **401/403**: token ausente o permisos de backoffice insuficientes.
- **404**: cuenta, organización o token no encontrado.
- Algunas operaciones de update/delete pueden devolver body vacío.
