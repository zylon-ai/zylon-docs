---
title: "Herramienta de web fetch"
description: "Descarga y extrae una URL mencionada en el mensaje."
---

Usa web fetch cuando tu usuario pregunte sobre una página. Incluye la URL en el mensaje, habilita la herramienta y el asistente puede descargar y extraer texto legible.

## Requisitos previos

```bash
export API_TOKEN="YOUR_API_TOKEN"
```

Si aún no tienes un token, consulta [Gestión de tokens](/es/api-reference/token-management).

## Requisitos

- Tool spec: `{ "name": "web_extract", "type": "web_extract_v1" }`
- No requiere `tool_context`

## Solicitud y respuesta básicas

```bash
curl -X POST "https://{base_url}/gpt/v1/messages/validate" \
  -H "Authorization: Bearer $API_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "default",
    "max_tokens": 128,
    "messages": [
      { "role": "user", "content": "Summarize: https://example.com/" }
    ],
    "tools": [
      { "name": "web_extract", "type": "web_extract_v1" }
    ],
    "tool_choice": { "type": "auto" }
  }'
```

Respuesta de ejemplo:

```json
{ "valid": true, "errors": null }
```

Para ejecutar la request, envía el mismo body a `POST /messages`.

## Consejos

- Sé explícito en el mensaje sobre lo que quieres (resumen, hechos clave, etc.).
- Si tu despliegue restringe salida a internet, esta herramienta puede estar deshabilitada.
