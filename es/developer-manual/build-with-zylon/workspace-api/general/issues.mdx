title: "Issues"

Los issues permiten reportar problemas con respuestas o contenido. Usa estos endpoints para listar, crear, actualizar y cerrar issues.

## Solicitud básica y respuesta

Lista issues con `GET /v1/app/issue` antes de crear o actualizar.

```bash
curl "https://{base_url}/v1/app/issue?page=1&page_size=20" \
  -H "Authorization: Bearer {api_token}"
```

## Crear un issue

```bash
curl -X POST "https://{base_url}/v1/app/issue" \
  -H "Authorization: Bearer {api_token}" \
  -H "Content-Type: application/json" \
  -d '{
    "description": "Response missed the attached policy details.",
    "resource_type": "Thread",
    "type": "NoContextUsed",
    "anonymous": false
  }'
```

## Obtener o actualizar un issue

<Tabs>
  <Tab title="Obtener issue">
```bash
curl "https://{base_url}/v1/app/issue/issue_5c4b3a2d1e0f9a8b" \
  -H "Authorization: Bearer {api_token}"
```

Ejemplo de respuesta (esquema no definido en OpenAPI):

```json
{
  "id": "issue_5c4b3a2d1e0f9a8b",
  "issue_number": 1842,
  "description": "Response missed the attached policy details.",
  "state": "NotReviewed"
}
```
  </Tab>
  <Tab title="Actualizar issue">
```bash
curl -X PUT "https://{base_url}/v1/app/issue/issue_5c4b3a2d1e0f9a8b" \
  -H "Authorization: Bearer {api_token}" \
  -H "Content-Type: application/json" \
  -d '{
    "state": "InReview",
    "is_read": true
  }'
```

Ejemplo de respuesta:

```json
{
  "id": "issue_5c4b3a2d1e0f9a8b",
  "state": "InReview",
  "is_read": true
}
```
  </Tab>
</Tabs>

## Eliminar un issue

```bash
curl -X DELETE "https://{base_url}/v1/app/issue/issue_5c4b3a2d1e0f9a8b" \
  -H "Authorization: Bearer {api_token}"
```

Ejemplo de respuesta:

```text
HTTP/1.1 204 No Content
```

## Errores y casos límite

- **404**: issue no encontrado.
- **400**: estado inválido.
