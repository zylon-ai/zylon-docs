<Update label="1.54.0" description="March 09, 2026">
  <Danger>
    **⚠️ Acción requerida antes de actualizar a la versión ≥ 1.54.0**

    Esta versión requiere migrar a la nueva versión de la CLI antes de actualizar. Omitir este paso causará un fallo crítico y Zylon no podrá iniciarse.
    Esta actualización requiere entre 45 y 60 minutos de inactividad planificada. Planifícalo con antelación y notifica a tus usuarios antes de comenzar.
    Esta versión contiene un cambio incompatible con versiones antiguas con los tokens de API del Workspace.

    Consulta la [guía de migración completa de la versión 1.54.0](/es/changelog/guides/1-54-0) para todos los detalles.
  </Danger>

  ## 💼 Workspace
  ### ✨ Funcionalidades

  - **Renovación de los registros de auditoría:** mayor visibilidad y capacidades de filtrado en las entradas de auditoría con seguimiento granular de las acciones de los usuarios.
  - **Conector de Confluence On-Prem:** integración con instancias on-premises de Confluence para un acceso y colaboración documental sin fricciones. [Más información](/es/user-manual/connectors/confluence-server)

  ### 🐛 Correcciones de errores

  - Se corrigieron cierres inesperados al procesar consultas SQL excesivamente grandes con el conector de base de datos.
  - Se añadieron títulos correctos para conectores MCP cuando solo hay una herramienta.
  - Se corrigió un error en chats cuando ocurría la compresión de contexto.
  - Se optimizó el selector de organización para listas largas, mejorando la usabilidad.

  ## 🛠️ Plataforma
  ### ✨ Funcionalidades

  - **Backoffice:** el Backoffice de la plataforma es utilizado por los operadores para gestionar el acceso a nivel de plataforma y los entornos de desarrollo. [Más información](/es/operator-manual/backoffice/introduction)
  - **AI Gateways:** un servicio de software dedicado y aislado que se sitúa entre los clientes y las APIs internas de Zylon. [Más información](/es/operator-manual/backoffice/platform-backoffice#gateways)
</Update>

<Update label="1.52.0" description="January 28, 2026">

  ## 💼 Workspace
  - **Soporte para DB2**: Ahora puedes conectar tus bases de datos DB2 directamente a Zylon, permitiendo que la IA razone sobre tus datos empresariales mientras mantienes control total y seguridad.
  - **Mejoras en integraciones MCP**: Ahora se admite autenticación con token Bearer para integraciones MCP, proporcionando opciones de seguridad más flexibles. También puedes activar o desactivar todas las herramientas MCP a la vez, facilitando la configuración de tu workspace.
  - **Selección de Carpeta de SharePoint**: Al usar el selector de SharePoint, ahora puedes establecer una carpeta de inicio predeterminada para toda la organización, ahorrando tiempo al navegar a ubicaciones frecuentemente utilizadas.
  - **Ventana de Analíticas Extendida**: Los datos de analíticas ahora están disponibles durante los últimos 12 meses, proporcionándote una vista más amplia del uso del workspace y las tendencias.
  - **Visibilidad del Conector**: Ahora puedes ver quién creó cada conector, mejorando la transparencia y facilitando la gestión de integraciones en tu equipo.
  - **Columna de Proyecto en Reporte de Problemas**: El rastreador de problemas ahora muestra una columna de proyecto con nombres de proyectos, facilitando la organización y el filtrado de problemas reportados.

  ## 🛠️ Plataforma
  - **Despliegues Privados de N8N**: Ahora puedes desplegar tu propio clúster privado de N8N dentro de tu infraestructura. Úsalo de forma independiente para construir flujos de trabajo personalizados, o crea tus propios servidores MCP para capacidades de IA extendidas. N8N maneja tu automatización personalizada mientras Zylon proporciona la capa de IA, asegurando que todos los datos permanezcan seguros dentro de tu entorno.

  ## 🐛 Correcciones de Errores
  - **Mejoras en redirección** en toda la plataforma.
  - **Estabilidad en Creación y Edición de Conectores**: Se corrigieron múltiples casos donde las operaciones de creación y edición de conectores fallaban inesperadamente.

<Update label="1.34.2" description="July 16, 2025">
  <Warning>
    Es esencial actualizar la aplicación web de Zylon inmediatamente al recibir la notificación de actualización, haciendo clic en el
    botón Actualizar. La aplicación no funcionará correctamente hasta que se complete la actualización. Si algún usuario tiene problemas, limpia
    la caché y actualiza.
  </Warning>
  ### ✨ Funcionalidades

  - **Selector de herramientas**: El usuario tiene la capacidad de seleccionar herramientas entre disponibles: Análisis numérico y Pensamiento
    extendido (solo disponible con Qwen 3).
  - Análisis numérico (actualmente en beta): Habilita operaciones estándar en datos tabulares en formato textual, incluyendo
    cálculos como promedios, filtrado, selección y visualización de datos usando gráficos. En la actualidad, esta
    funcionalidad está disponible solo cuando se selecciona un archivo CSV de la base de conocimiento. Para un rendimiento óptimo, recomendamos
    adherirse a las siguientes directrices:
  - Se debe incluir una sola fila de encabezado.
  - Las fechas deben formatearse como yyyy-mm-dd.

  **Cambios requeridos debido al nuevo marco de métricas de observabilidad**

  Para proporcionar mejor soporte en caso de que algo salga mal, en esta versión hemos actualizado cómo funciona el
  marco de observabilidad incluyendo una nueva herramienta basada en Grafana, más allá del mecanismo de reporte de fallos que teníamos
  con Sentry. Para que funcione, por favor asegúrate de que tu firewall no esté bloqueando conexiones a cualquiera de estos sitios web:

  - sentry.io (deberías tener esto ya configurado correctamente)
  - \*.grafana.net (este es el nuevo requerido)

  Recuerda que cualquier dato que enviamos es completamente anónimo y agregado, y nos ayudará a diagnosticar cualquier
  mal funcionamiento en la plataforma y entender cómo se está usando Zylon para acelerar acciones de soporte.

  Para aquellos que quieran saber más sobre lo que ha cambiado, hemos evolucionado el archivo de configuración antiguo:

  ```yaml
  sentry:
  enabled: true
  environment: "my-company"
  ```

  a este nuevo y más granular:

  ```yaml
  observability:
  crashReporting: true
  usageMetrics: true
  
  companyIdentifier: "my-company"
  ```

  En caso de que quieras optar por no participar en esto, puedes añadir el siguiente fragmento al archivo de configuración, pero recuerda que
  hacerlo nos dejará ciegos contra cualquier acción de soporte de nuestra parte:

  ```yaml
  observability:
  crashReporting: false
  usageMetrics: false
  ```
