---
title: "Semantic Search tool"
description: "Busca en artefactos ingeridos durante un mensaje."
---

Usa Semantic Search cuando quieras recuperar información de artefactos ingeridos (documentos, notas, PDFs, HTML, etc.).

## Requisitos previos

- Un token de API. Ver [Gestión de tokens](/es/api-reference/token-management).
- Artefactos ingeridos para buscar.

## Requisitos

- Tool spec: `{ "name": "semantic_search", "type": "semantic_search_v1" }`
- `tool_context`: al menos un `ingested_artifact` con `context_filter`

## Solicitud y respuesta básicas

Este ejemplo valida una request que habilita semantic search:

```bash
curl -X POST "https://{base_url}/gpt/v1/messages/validate" \
  -H "Authorization: Bearer {api_token}" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "default",
    "max_tokens": 128,
    "messages": [
      { "role": "user", "content": "What shipped in the latest billing dashboard release?" }
    ],
    "tools": [
      { "name": "semantic_search", "type": "semantic_search_v1" }
    ],
    "tool_choice": { "type": "auto" },
    "tool_context": [
      {
        "type": "ingested_artifact",
        "context_filter": {
          "collection": "docs_examples",
          "artifacts": ["docs_file_artifact"]
        }
      }
    ]
  }'
```

Ejemplo de respuesta:

```json
{ "valid": true, "errors": null }
```

Para ejecutar, envía el mismo body a `POST /messages`.

## Tips

- Mantén `tool_context` acotado.
- Si los resultados no son relevantes, ingesta texto más limpio.
