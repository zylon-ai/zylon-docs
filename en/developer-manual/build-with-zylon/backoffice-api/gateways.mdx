title: "Gateways"

A **Gateway** is an isolated runtime boundary for API traffic in Zylon. It is not a hardware device.

Gateways are used to:

- isolate API environments by team or use case
- control who can issue and use tokens in each environment
- scope data access and activity through gateway-level policies

## Prerequisites

- Backoffice-capable token in `Authorization: Bearer {API_TOKEN}`
- Target base URL (`https://{BASE_URL}`)

## Gateways

### Create a gateway

```bash
curl -X POST "https://{BASE_URL}/api/v1/platform/gateway" \
  -H "Authorization: Bearer {API_TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "docs-gw-final",
    "description": "Docs final examples",
    "interceptors": []
  }'
```

## Gateway tokens

Gateway tokens are intended for Workspace API access.

### Create a gateway token

```bash
curl -X POST "https://{BASE_URL}/api/v1/platform/gateway/{gatewayId}/token" \
  -H "Authorization: Bearer {API_TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "docs-gw-token-final",
    "scopes": ["Workspace.ReadWrite.All"],
    "valid_until": "2027-12-31T00:00:00Z",
    "revoke_existing": false
  }'
```

<Accordion title="Example response on demo (400)">
```json
{
  "type": "BadRequest",
  "status": 400,
  "message": "At least one valid scope must be specified for the API token"
}
```
</Accordion>

### List gateway tokens

```bash
curl "https://{BASE_URL}/api/v1/platform/gateway/{gatewayId}/token?page_size=2" \
  -H "Authorization: Bearer {API_TOKEN}"
```

<Accordion title="Example response">
```json
{
  "data": [],
  "has_next_page": false,
  "has_previous_page": false
}
```
</Accordion>

### Update a gateway token

```bash
curl -X PUT "https://{BASE_URL}/api/v1/platform/gateway/{gatewayId}/token/{token_hash}" \
  -H "Authorization: Bearer {API_TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{"name": "docs-gw-token-final-rotated"}'
```

<Accordion title="Example response">
```text
No response body is currently documented for this endpoint.
Use HTTP 2xx as success signal.
```
</Accordion>

### Delete a gateway token

```bash
curl -X DELETE "https://{BASE_URL}/api/v1/platform/gateway/{gatewayId}/token/{token_hash}" \
  -H "Authorization: Bearer {API_TOKEN}"
```

<Accordion title="Example response (204)">
```text
No response body.
```
</Accordion>

## Common mistakes

- Omitting `interceptors` in create/update gateway payloads.
- Assuming every scope string is valid for gateway token creation.
- Mixing account IDs and user IDs in gateway user operations.

## Errors and edge cases

- **400**: invalid request body or unsupported token scope.
- **404**: gateway or gateway user not found.
- **204**: successful token or gateway deletes can return no body.
