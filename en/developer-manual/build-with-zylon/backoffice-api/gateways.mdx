title: "Gateways"

A **Gateway** is an isolated software node for API development in Zylon. It is not a hardware device.

Gateways are used to:

- group API development activity in separate environments
- control who can work with tokens in each environment
- log account and token usage for that gateway

## Prerequisites

- Backoffice-capable token in `Authorization: Bearer {API_TOKEN}`
- Target base URL (`https://{BASE_URL}`)

## Gateways

### Create a gateway

```bash
curl -X POST "https://{BASE_URL}/api/v1/platform/gateway" \
  -H "Authorization: Bearer {API_TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Support Gateway",
    "description": "Gateway for support operations",
    "use_default_interceptors": true
  }'
```

<Accordion title="Example response">
```json
{
  "id": "gw_9a2e9f4f",
  "name": "Support Gateway",
  "description": "Gateway for support operations",
  "type": "Custom",
  "gateway_users": []
}
```
</Accordion>

### List gateways

`GET /api/v1/platform/gateway` supports pagination (`page`, `page_size`, `page_order`, `after`, `before`, `first`, `last`, `total_count`).

```bash
curl "https://{BASE_URL}/api/v1/platform/gateway?page_size=20&page=1" \

```

## Gateway tokens

Gateway tokens are intended for Workspace API access.

### Create a gateway token

```bash
curl -X POST "https://{BASE_URL}/api/v1/platform/gateway/{gatewayId}/token" \
  -H "Authorization: Bearer {API_TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "ci-token",
    "valid_until": "2027-12-31T00:00:00Z",
    "revoke_existing": false
  }'
```

<Accordion title="Example response">
```json
{
  "name": "ci-token",
  "token": "v1--newly-issued-token",
  "hashed_token": "sha256:2c1f...",
  "token_email": "sasha@auroralabs.com",
  "gateway_id": "gw_9a2e9f4f",
  "last_used_at": "",
  "created_at": "2026-02-08T14:20:00Z",
  "valid_until": "2027-12-31T00:00:00Z"
}
```
</Accordion>

### List gateway tokens

```bash
curl "https://{BASE_URL}/api/v1/platform/gateway/{gatewayId}/token?page_size=20" \
  -H "Authorization: Bearer {API_TOKEN}"
```

<Accordion title="Example response">
```json
{
  "data": [
    {
      "name": "ci-token",
      "hashed_token": "sha256:2c1f...",
      "token_email": "sasha@auroralabs.com",
      "gateway_id": "gw_9a2e9f4f",
      "last_used_at": "2026-02-09T11:43:00Z",
      "created_at": "2026-02-08T14:20:00Z",
      "valid_until": "2027-12-31T00:00:00Z"
    }
  ],
  "has_next_page": false,
  "has_previous_page": false,
  "total_count": 1
}
```
</Accordion>

### Update a gateway token

```bash
curl -X PUT "https://{BASE_URL}/api/v1/platform/gateway/{gatewayId}/token/{token_hash}" \
  -H "Authorization: Bearer {API_TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{"name": "ci-token-rotated"}'
```

<Accordion title="Example response">
```text
No response body is currently documented for this endpoint.
Use HTTP 2xx as success signal.
```
</Accordion>

### Delete a gateway token

```bash
curl -X DELETE "https://{BASE_URL}/api/v1/platform/gateway/{gatewayId}/token/{token_hash}" \
  -H "Authorization: Bearer {API_TOKEN}"
```

<Accordion title="Example response (204)">
```text
No response body.
```
</Accordion>

## Common mistakes

- Mixing account IDs and user IDs in gateway user operations.
- Rotating tokens without storing the new token value from create responses.

## Errors and edge cases

- **400**: invalid body when adding/updating gateway users.
- **404**: gateway or gateway user not found.
- **204**: successful token or gateway deletes can return no body.
