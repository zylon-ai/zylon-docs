---
title: "Project Members"
description: "Add, update, and remove members from a project."
---

Use project member endpoints to control who can access a project and what role they have. Roles include `Viewer`, `Commenter`, `Editor`, `Admin`, and `Owner`.

## Basic request and response

List current members with `GET /v1/app/project/{projectId}/member`.

```bash
curl "https://{BASE_URL}/v1/app/project/proj_7a5c3e1b9d2f4a6c/member?page=1&page_size=50" \
  -H "Authorization: Bearer {API_TOKEN}"
```

## Add members

```bash
curl -X POST "https://{BASE_URL}/v1/app/project/proj_7a5c3e1b9d2f4a6c/member" \
  -H "Authorization: Bearer {API_TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{
    "members": [
      { "user_id": "user_9d1c3b5a7f2e4c6d", "role": "Editor" }
    ]
  }'
```

<Accordion title="Example response">

## Update member roles

```bash
curl -X PUT "https://{BASE_URL}/v1/app/project/proj_7a5c3e1b9d2f4a6c/member" \
  -H "Authorization: Bearer {API_TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{
    "members": [
      { "user_id": "user_9d1c3b5a7f2e4c6d", "role": "Admin", "is_pinned": true }
    ]
  }'
```

## Errors and edge cases

- **403**: not allowed to manage members.
- **404**: project or user not found.
