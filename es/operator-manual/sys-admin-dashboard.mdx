Zylon viene con un [Kubernetes Dashboard](https://kubernetes.io/docs/tasks/access-application-cluster/web-ui-dashboard/) integrado,
que te ayudará a monitorear y diagnosticar problemas con tu instalación.

Para habilitarlo, necesitarás editar tu archivo de configuración en `/etc/zylon/zylon-conf.yaml` y agregar la siguiente sección:
```yaml
adminDashboard:
  enabled: true
  exposeIngress: true
```

A continuación, actualiza tu sistema Zylon ejecutando:
```bash
sudo zylon-cli update
```

Una vez que se complete la actualización, accede al panel en:
`https://<zylon.company.com>/kube-dashboard`

Para iniciar sesión, puedes generar un token de corta duración:
```bash
kubectl -n zylon create token zylon
```
O un token permanente:
```bash
kubectl get secret zylon-token -n zylon -o jsonpath='{.data.token}' | base64 -d
```

<Warning>
    El token permanente proporcionará acceso completo de administrador a tu clúster de Kubernetes.
    Manténlo secreto y seguro.
</Warning>

<Tip>
    Zylon usa múltiples namespaces, puedes verificar el estado de cada uno.
    Selecciona el namespace correcto del menú desplegable en la esquina superior izquierda.

    * zylon: Componentes del sistema Zylon
    * openebs: Sistema de almacenamiento OpenEBS
    * nvidia: Controlador NVIDIA y plugin de dispositivo
</Tip>


Después de iniciar sesión, podrás ver el estado de tu instalación de Zylon, incluyendo:
- Pods
- Logs
- Métricas básicas sobre el uso de CPU y memoria

## Preguntas frecuentes

### ❓ ¿Por dónde empiezo cuando algo no funciona?

Cuando abres el panel de Kubernetes por primera vez, verás una vista general del estado de tus cargas de trabajo y del clúster.
Esta vista muestra el estado actual de los recursos en ejecución, como Pods, Deployments y el uso básico de CPU y memoria.

![image](/images/operator-manual/k8s-1.png)

---

### ❓ ¿Qué namespace debo seleccionar?

Zylon utiliza varios namespaces, pero la mayoría de los problemas relacionados con workspaces y la plataforma estarán en el namespace `zylon`.

1. Abre el desplegable de namespaces en la **esquina superior izquierda** del dashboard
2. Selecciona el namespace que quieras revisar
3. El dashboard se actualizará y mostrará únicamente los componentes del sistema de ese namespace

Esto ayuda a reducir el ruido y asegura que estás inspeccionando los recursos correctos.

![image](/images/operator-manual/k8s-2.png)
---

### ❓ ¿Cómo puedo ver los logs de un Pod?

Para inspeccionar los logs de un Pod concreto:

1. Haz clic en el **nombre del Pod** que quieres investigar
2. Esto abrirá la página de detalles del Pod, donde podrás ver:
- Estado
- Eventos
- Uso de recursos
- Contenedores
3. En la **esquina superior derecha**, haz clic en **Logs**

![image](/images/operator-manual/k8s-3.png)

Desde aquí puedes:
- Ver los logs en tiempo real
- Descargar los logs para analizarlos offline si tu instancia está en modo *airgap*

Los logs suelen ser la forma más rápida de identificar errores de configuración, fallos durante el arranque o problemas en tiempo de ejecución.
