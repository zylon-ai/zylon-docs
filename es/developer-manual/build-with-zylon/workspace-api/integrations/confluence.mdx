Las integraciones de Confluence importan páginas y espacios para usarlos como artefactos. **Debes habilitar Confluence a nivel de instancia primero**—consulta la guía del operador: [Configuración de Confluence](/es/operator-manual/knowledge-base/confluence).

Si la integración requiere OAuth, usa los endpoints de [OpenID y OAuth](/es/developer-manual/build-with-zylon/workspace-api/authentication/openid).

## Solicitud básica y respuesta

Obtén la configuración con `GET /api/v1/app/integration/confluence/{integrationId}/config`.

```bash
curl "https://{BASE_URL}/api/v1/app/integration/confluence/{integrationID}/config" \
  -H "Authorization: Bearer {API_TOKEN}"
  -H "x-org: {org_slug}"
```

<Accordion title="Ejemplo de respuesta">
```json
{
  "space_key": "SUP",
  "base_url": "https://confluence.auroralabs.com"
}
```
</Accordion>

## Iniciar sincronización

```bash
curl -X POST "https://{BASE_URL}/api/v1/app/integration/confluence/{integrationID}/ingest" \
  -H "Authorization: Bearer {API_TOKEN}" \
  -H "x-org: {org_slug}" \
  -H "Content-Type: application/json" \
  -d '{}'
```

<Accordion title="Ejemplo de respuesta">
```json
{
  "type": "sync_started",
  "current_artifact_count": 42,
  "integration_item_count": 120
}
```
</Accordion>

## Endpoints ConfluenceLocalLegacy

Usa estos endpoints cuando tu despliegue utilice el proveedor `confluencelocallegacy` con credenciales basic-auth.

### Crear una integración ConfluenceLocalLegacy

```bash
curl -X POST "https://{BASE_URL}/api/v1/app/integration/confluencelocallegacy" \
  -H "Authorization: Bearer {API_TOKEN}" \
  -H "x-org: {org_slug}" \
  -H "Content-Type: application/json" \
  -d '{
    "username": "confluence_service",
    "password": "{password}"
  }'
```

### Actualizar una integración ConfluenceLocalLegacy

```bash
curl -X POST "https://{BASE_URL}/api/v1/app/integration/confluencelocallegacy/{integrationID}" \
  -H "Authorization: Bearer {API_TOKEN}" \
  -H "x-org: {org_slug}" \
  -H "Content-Type: application/json" \
  -d '{
    "username": "confluence_service",
    "password": "{password}"
  }'
```

### Obtener la configuración ConfluenceLocalLegacy

```bash
curl "https://{BASE_URL}/api/v1/app/integration/confluencelocallegacy/{integrationID}/config" \
  -H "Authorization: Bearer {API_TOKEN}"
  -H "x-org: {org_slug}"
```

### Iniciar ingest de ConfluenceLocalLegacy

```bash
curl -X POST "https://{BASE_URL}/api/v1/app/integration/confluencelocallegacy/{integrationID}/ingest" \
  -H "Authorization: Bearer {API_TOKEN}" \
  -H "x-org: {org_slug}" \
  -H "Content-Type: application/json" \
  -d '{}'
```

<Accordion title="Ejemplo de respuesta de integración">
```json
{
  "id": "int_6d9a2b4c5e7f",
  "user_id": "user_4b7c2a1d9e5f3c8b",
  "org_id": "org_2f3a9d1c7b5e4a8f",
  "type": "ConfluenceLocalLegacy",
  "provider_data": {
    "username": "confluence_service"
  },
  "created_at": "2026-03-01T11:00:00Z",
  "updated_at": "2026-03-01T11:00:00Z"
}
```
</Accordion>

## Errores comunes

- Usar endpoints de Confluence con OAuth sobre IDs de integración `confluencelocallegacy`.
- Omitir campos JSON (`username`, `password`) en crear/actualizar.
- Lanzar la ingestión antes de guardar la configuración de la integración.

## Errores y casos límite

- **403**: integración no habilitada o sin permisos.
- **404**: integración no encontrada.
