---
title: "Visión general"
description: "Permite que Zylon use herramientas durante una respuesta."
---

Las herramientas permiten que el asistente ejecute operaciones específicas (como buscar artefactos, analizar datos tabulares o extraer una página web) como parte de una respuesta.

Puedes habilitar herramientas añadiendo tres campos a una request `POST /messages`:

- `tools`: qué herramientas están permitidas
- `tool_choice`: cómo debe elegir herramientas el asistente (`auto`, forzar una herramienta o `none`)
- `tool_context`: contexto opcional (documentos o conexiones de BD)

## Requisitos previos

- Un token de API. Ver [Gestión de tokens](/es/api-reference/token-management).
- Tu hostname de Zylon (reemplaza `{base_url}` en los ejemplos).

## Solicitud y respuesta básicas

Empieza con `POST /messages/validate` para validar el body sin ejecutar el modelo.

Este ejemplo habilita **web fetch**:

```bash
curl -X POST "https://{base_url}/gpt/v1/messages/validate" \
  -H "Authorization: Bearer {api_token}" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "default",
    "max_tokens": 64,
    "messages": [
      { "role": "user", "content": "Summarize: https://example.com/" }
    ],
    "tools": [
      { "name": "web_extract", "type": "web_extract_v1" }
    ],
    "tool_choice": { "type": "auto" }
  }'
```

Ejemplo de respuesta:

```json
{ "valid": true, "errors": null }
```

Para ejecutar la request, envía el mismo body a `POST /messages`.

## Herramientas built-in

Zylon incluye herramientas built-in:

| Tool type | Qué hace | Requiere `tool_context` |
| --- | --- | --- |
| `semantic_search_v1` | Buscar artefactos ingeridos | Sí (`ingested_artifact`) |
| `tabular_analysis_v1` | Analizar artefactos tabulares | Sí (`ingested_artifact`) |
| `web_extract_v1` | Extraer una URL del mensaje | No |
| `web_search_v1` | Buscar en la web | No |
| `database_query_v1` | Consultar una BD SQL | Sí (`sql_database`) |

## Errores comunes

- **Nombre de campo**: usa `inputSchema` (no `input_schema`).
- **Falta de contexto**: algunas herramientas requieren `tool_context`.
- **Auth**: incluye `Authorization: Bearer ...` en cada request.
