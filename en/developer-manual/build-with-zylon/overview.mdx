Most ZylonGPT applications follow the same loop:

1. **Ingest knowledge** you want the assistant to use (Artifacts).
2. **Generate a response** (Messages).
3. **Retrieve or analyze** that knowledge (Tools).

This section focuses on the pieces youâ€™ll combine to build real features.

## Messages

The Messages API is how you talk to the model: you send a conversation history, and you receive the next assistant message back as structured content blocks.

Start here: [Messages API](/en/developer-manual/build-with-zylon/working-with-messages)

## Artifacts

Artifacts are how you persist context in Zylon (text, files, or URIs) inside a collection, so it can be retrieved and reused across requests.

Start here: [Artifacts API](/en/developer-manual/build-with-zylon/working-with-artifacts)

## Tools

Tools let Zylon run targeted operations (like searching artifacts, analyzing tabular data, or extracting a web page) as part of a message response.

Start here: [Tools](/en/developer-manual/build-with-zylon/tools/introduction-to-tools)
