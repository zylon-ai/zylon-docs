---
title: "Project Members"
description: "Add, update, and remove members from a project."
---

Use project member endpoints to control who can access a project and what role they have. Roles include `Viewer`, `Commenter`, `Editor`, `Admin`, and `Owner`.

## Basic request and response

List current members with `GET /api/v1/app/project/{projectId}/member`.

<Accordion title="Endpoint parameters">
| Type | Name | Required | Notes |
| --- | --- | --- | --- |
| Path | `projectID` | Yes | Project identifier. |
| Query | `page` | No | Absolute page number to fetch, 1-based. |
| Query | `page_size` | No | Number of items per page. Default is `20`. |
| Query | `page_order` | No | `ASC` or `DESC`. Default is `ASC`. |
| Query | `after` | No | Cursor or value to paginate after. |
| Query | `before` | No | Cursor or value to paginate before. |
| Query | `first` | No | Limit to the first N items after a cursor. |
| Query | `last` | No | Limit to the last N items before a cursor. |
| Query | `total_count` | No | Include total count in the response. |
</Accordion>

```bash
curl "https://{BASE_URL}/api/v1/app/project/{projectID}/member?page=1&page_size=50" \
  -H "Authorization: Bearer {API_TOKEN}"
  -H "x-org: {org_slug}"
```

## Add members

<Accordion title="Endpoint parameters">
| Type | Name | Required | Notes |
| --- | --- | --- | --- |
| Path | `projectID` | Yes | Project identifier. |
| Body | `members` | Yes | Array of members to add. |
| Body | `members[].user_id` | Yes | User identifier. |
| Body | `members[].role` | Yes | `Viewer`, `Commenter`, `Editor`, `Admin`, or `Owner`. |
</Accordion>

```bash
curl -X POST "https://{BASE_URL}/api/v1/app/project/{projectID}/member" \
  -H "Authorization: Bearer {API_TOKEN}" \
  -H "x-org: {org_slug}" \
  -H "Content-Type: application/json" \
  -d '{
    "members": [
      { "user_id": "{userID}", "role": "Editor" }
    ]
  }'
```

<Accordion title="Example response">
```json

  {

      "account_id": "acct_6c8a1f3b2d4e5f7a",
```

## Update member roles

<Accordion title="Endpoint parameters">
| Type | Name | Required | Notes |
| --- | --- | --- | --- |
| Path | `projectID` | Yes | Project identifier. |
| Body | `members` | Yes | Array of members to update. |
| Body | `members[].user_id` | Yes | User identifier. |
| Body | `members[].role` | No | `Viewer`, `Commenter`, `Editor`, `Admin`, or `Owner`. |
| Body | `members[].is_pinned` | No | Pins or unpins the project for that member. |
</Accordion>

```bash
curl -X PUT "https://{BASE_URL}/api/v1/app/project/{projectID}/member" \
  -H "Authorization: Bearer {API_TOKEN}" \
  -H "x-org: {org_slug}" \
  -H "Content-Type: application/json" \
  -d '{
    "members": [
      { "user_id": "{userID}", "role": "Admin", "is_pinned": true }
    ]
  }'
```

```json

      "account_id": "acct_6c8a1f3b2d4e5f7a",
```

## Errors and edge cases

- **403**: not allowed to manage members.
- **404**: project or user not found.
