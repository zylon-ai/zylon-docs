---
title: "Organizations"
description: "List and update organizations within your workspace."
---

Organizations are the top-level container for users and projects. Use these endpoints to list orgs visible to your account and update org-level settings.

## Basic request and response

List organizations visible to your account with `GET /v1/app/organization`.

```bash
curl "https://{base_url}/v1/app/organization" \
  -H "Authorization: Bearer {api_token}"
```

Example response (schema not defined in OpenAPI):

```json
[
  {
    "name": "Aurora Labs",
    "context": "Enterprise R&D and analytics.",
    "email_regex": ".*@auroralabs.com"
  }
]
```

## Update organization settings

```bash
curl -X POST "https://{base_url}/v1/app/organization" \
  -H "Authorization: Bearer {api_token}" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Aurora Labs",
    "context": "Private AI platform for product and support teams.",
    "email_regex": ".*@auroralabs.com"
  }'
```

Example response (schema not defined in OpenAPI):

```text
HTTP/1.1 200 OK
```

## Errors and edge cases

- **401/403**: missing or invalid API token.
- **400**: invalid regex or payload shape.
