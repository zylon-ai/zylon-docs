---
title: "Organizaciones"
description: "Lista y actualiza organizaciones dentro del workspace."
---

Las organizaciones son el contenedor principal de usuarios y proyectos. Usa estos endpoints para listar organizaciones visibles y actualizar configuración a nivel org.

## Solicitud básica y respuesta

Lista organizaciones visibles con `GET /v1/app/organization`.

```bash
curl "https://{base_url}/v1/app/organization" \
  -H "Authorization: Bearer {api_token}"
```

Ejemplo de respuesta (esquema no definido en OpenAPI):

```json
[
  {
    "name": "Aurora Labs",
    "context": "Enterprise R&D and analytics.",
    "email_regex": ".*@auroralabs.com"
  }
]
```

## Actualizar configuración de organización

```bash
curl -X POST "https://{base_url}/v1/app/organization" \
  -H "Authorization: Bearer {api_token}" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Aurora Labs",
    "context": "Private AI platform for product and support teams.",
    "email_regex": ".*@auroralabs.com"
  }'
```

Ejemplo de respuesta (esquema no definido en OpenAPI):

```text
HTTP/1.1 200 OK
```

## Errores y casos límite

- **401/403**: token inválido o ausente.
- **400**: regex o payload inválido.
