---
title: "Auditoría"
description: "Consulta auditorías a nivel workspace o proyecto."
---

Los endpoints de auditoría permiten revisar acciones con filtros por usuario, acción y rango de tiempo.

## Solicitud básica y respuesta

Consulta el log del workspace con `GET /v1/app/audit`.

```bash
curl "https://{base_url}/v1/app/audit?start_date=1738291200&end_date=1738972800&include=User&page=1&page_size=50" \
  -H "Authorization: Bearer {api_token}"
```

Ejemplo de respuesta (esquema no definido):

```json
[]
```

## Auditoría de un proyecto

```bash
curl "https://{base_url}/v1/app/project/proj_7a5c3e1b9d2f4a6c/audit?start_date=1738291200&end_date=1738972800&page=1&page_size=50" \
  -H "Authorization: Bearer {api_token}"
```

Ejemplo de respuesta (esquema no definido):

```json
[]
```

## Errores y casos límite

- **400**: filtros o rango de fechas inválido.
- **403**: permisos insuficientes.
