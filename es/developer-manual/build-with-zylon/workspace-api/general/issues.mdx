title: "Incidencias"
description: "Crea y gestiona incidencias comunicadas por usuarios."
---

Las incidencias (`issues`) permiten comunicar problemas con respuestas o contenido. Usa estos endpoints para listar, crear, actualizar y cerrar incidencias.

## Solicitud básica y respuesta

Lista incidencias con `GET /v1/app/issue` antes de crear o actualizar.

```bash
curl "https://{BASE_URL}/v1/app/issue?page=1&page_size=20" \
  -H "Authorization: Bearer {API_TOKEN}"
```

## Crear una incidencia

```bash
curl -X POST "https://{BASE_URL}/v1/app/issue" \
  -H "Authorization: Bearer {API_TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{
    "description": "Response missed the attached policy details.",
    "resource_type": "Thread",
    "type": "NoContextUsed",
    "anonymous": false
  }'
```

## Obtener o actualizar una incidencia

<Tabs>
  <Tab title="Obtener incidencia">
```bash
curl "https://{BASE_URL}/v1/app/issue/issue_5c4b3a2d1e0f9a8b" \
  -H "Authorization: Bearer {API_TOKEN}"
```

<Accordion title="Ejemplo de respuesta">
```json
{
  "id": "issue_5c4b3a2d1e0f9a8b",
  "issue_number": 1842,
  "description": "Response missed the attached policy details.",
  "state": "NotReviewed"
}
```
</Accordion>
  </Tab>
  <Tab title="Actualizar incidencia">
```bash
curl -X PUT "https://{BASE_URL}/v1/app/issue/issue_5c4b3a2d1e0f9a8b" \
  -H "Authorization: Bearer {API_TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{
    "state": "InReview",
    "is_read": true
  }'
```

<Accordion title="Ejemplo de respuesta">
```json
{
  "id": "issue_5c4b3a2d1e0f9a8b",
  "state": "InReview",
  "is_read": true
}
```
</Accordion>
  </Tab>
</Tabs>

## Eliminar una incidencia

```bash
curl -X DELETE "https://{BASE_URL}/v1/app/issue/issue_5c4b3a2d1e0f9a8b" \
  -H "Authorization: Bearer {API_TOKEN}"
```

<Accordion title="Ejemplo de respuesta">
```text
HTTP/1.1 204 No Content
```
</Accordion>

## Errores y casos límite

- **404**: incidencia no encontrada.
- **400**: estado inválido.
