## Overview
The Platform Backoffice is used by Operators to manage platform-level access and developer environments.

From this section, Operators can:

- Create and manage accounts
- Assign and update account roles
- Create and manage gateways
- Monitor platform service status

Only users with **Operator** permissions can access this section. Permissions, visibility, and limits depend on the active license.

## Accounts
An **Account** represents one person on the Zylon platform. Accounts define platform access and determine which Backoffice sections are available.

### Account roles
| Role | Primary use | Backoffice access |
| --- | --- | --- |
| **Super Admin** | Initial administrative account | Same platform access as Operator |
| **Operator** | Platform administration | Full access to Platform Backoffice |
| **Developer** | API development workflows | Access to Developer Tools |
| **Workspace** | Workspace collaboration | No Platform Backoffice access |

A single account can include more than one role.

### Roles and permissions
In Zylon, access control follows this model:

1. **Account role** grants platform-level access (for example, Operator or Developer).
2. **Context role** grants permissions inside a specific Workspace or Gateway.
3. Effective permissions are the combination of both.

For example, an account can be an **Operator** at platform level and also be a **Gateway Member** in a specific Gateway.

### Create an account
You can create users in two ways.

1. **Create from the modal UI**
- Click **Add account**.
- Enter name (optional), email, and password.
- Assign one or more account roles.
- Save the account.

2. **Create in bulk from CSV**
- Download the CSV template available on the Accounts page.
- Fill the template with user details and role assignments.
- Upload the CSV to create multiple accounts.

### Edit accounts and role assignments
To update an account, open the account from the list and modify its role assignments or details.

Use role assignment to control whether a user can:

- access Workspace,
- access Developer Tools,
- administer the platform.

The number of accounts and role-enabled accounts is controlled by the active license.

## Gateways
A **Gateway** is an isolated software node used for API development in Zylon. It is not a hardware device.

Gateways are used to:

- Group API development activity in separate environments
- Control who can work with tokens in each environment
- Log account usage and token usage for that gateway

### Gateway user roles
Accounts with the **Developer** role (and Operators who are added to a gateway) can be assigned a gateway-specific role:

| Gateway role | Capabilities |
| --- | --- |
| **Owner** | Manage gateway users and roles, create and manage tokens |
| **Admin** | Manage gateway users, create and manage tokens |
| **Member** | Create and manage own tokens |

### Create a gateway
To create a gateway:

1. Open **Gateways**.
2. Click **New gateway**.
3. Enter a gateway name.
4. Click **Create**.

### Manage gateways
From the gateway list, Operators can open existing gateways to manage users and maintain the gateway configuration used by development teams.

The total number of gateways is limited by the active license.
