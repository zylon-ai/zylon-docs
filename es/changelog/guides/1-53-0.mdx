<Warning>
  Esta guía es **obligatoria** antes de actualizar a Zylon 1.53.0 o cualquier versión posterior. Omitir estos pasos causará que Triton falle al arrancar.
</Warning>

## Qué cambió

Zylon 1.53.0 introduce dos cambios importantes en la plataforma que afectan cómo instalas y actualizas tu instancia:

**Todas las dependencias están incluidas como parte de la versión**: A partir de 1.53.0, cada versión de Zylon incluye sus propias dependencias empaquetadas, incluyendo los paquetes Apt, binarios y controladores necesarios para ejecutar esa versión. Esto significa que cuando instalas una versión específica de Zylon, obtienes las dependencias exactas necesarias para esa versión sin depender del estado global del sistema.

**Nueva versión, nuevos requisitos**: Zylon 1.53.0 requiere CUDA 13.0, que es un cambio significativo respecto a versiones anteriores. Durante el proceso de actualización, el CLI manejará automáticamente la actualización de CUDA. Esto será requerido para versiones más recientes, por lo que decidimos hacer este cambio importante ahora para evitar una actualización más disruptiva más adelante.

---

## Antes de comenzar

- Realiza una [copia de seguridad de tu instancia](/es/operator-manual/operations/backup) antes de comenzar.
- Planifica aproximadamente **45–60 minutos** de tiempo total de actualización.
- Espera **~35 minutos de tiempo de inactividad de GPU** durante el cual la inferencia no estará disponible.
- Notifica a tus usuarios antes de comenzar — el sistema estará parcialmente no disponible.

## Preguntas frecuentes

<AccordionGroup>
  <Accordion title="¿Puedo seguir usando 'sudo zylon-cli update'?">
    El comando sigue existiendo pero ya no gestiona las actualizaciones de versión a partir de 1.53.0. Siempre usa `sudo zylon-cli install <version>` para instalar o actualizar Zylon en adelante.
    Ahora, esto actuará como `sudo zylon-cli sync`.
  </Accordion>

  <Accordion title="¿Se verán afectados mis datos?">
    No. La actualización solo afecta la capa de plataforma y los controladores. Tu base de conocimiento, chats, agent flows y configuración no se tocan. Aún así recomendamos realizar una [copia de seguridad](/es/operator-manual/operations/backup) como precaución.
  </Accordion>

  <Accordion title="¿Qué sucede si omito la auto-actualización del CLI?">
    **Si Triton falla al arrancar después de actualizar**, la causa más probable es que el CLI no se actualizó antes de ejecutar la instalación. Puedes confirmarlo verificando la versión del CLI:

    ```bash
    sudo zylon-cli version
    ```

    Y verificando si Triton está ejecutándose:

    ```bash
    sudo kubectl get pods -A | grep triton
    ```

    Si el CLI es anterior a 2.0.0 y Triton no muestra 1/1 READY, entonces probablemente te encontraste con este problema. Para recuperar tu instalación, simplemente ejecuta la auto-actualización del CLI seguida de la instalación nuevamente:

    ```bash
    sudo zylon-cli self-update
    sudo zylon-cli install 1.53.0
    ```
  </Accordion>
</AccordionGroup>

## Pasos de migración

<Steps>
  <Step title="Actualizar el CLI">
    Ejecuta la auto-actualización del CLI para obtener Zylon CLI 2.0.0:

    ```bash
    sudo zylon-cli self-update
    ```

    Verifica la versión antes de continuar:

    ```bash
    sudo zylon-cli version
    # Esperado: 2.0.0 o superior
    ```

    <Warning>
      No procedas al siguiente paso hasta que el CLI esté en la versión 2.0.0. Instalar 1.53.0 con un CLI anterior resultará en una instalación defectuosa.
    </Warning>
  </Step>

  <Step title="Instalar la nueva versión">
    Usa el nuevo comando `install` para actualizar Zylon:

    ```bash
    sudo zylon-cli install 1.5X.0
    ```

    Esto instalará automáticamente toda la plataforma Zylon, incluyendo las dependencias, y la nueva versión.
    En versiones futuras, el CLI también manejará sus propias actualizaciones, por lo que ya no tendrás que ejecutar `self-update` por separado.
    
    <Warning>
      La instalación se detendrá con un mensaje de error después de desinstalar los controladores NVIDIA existentes:
      
      ```
      The current drivers were uninstalled successfully, please REBOOT the system 
      and re-run the installation to complete the NVIDIA driver installation
      ```
      
      **Este es un comportamiento esperado.** Procede al siguiente paso para reiniciar tu sistema, luego vuelve a ejecutar el comando de instalación.
    </Warning>
  </Step>

  <Step title="Reiniciar el sistema">
    Después de ver el mensaje de desinstalación, reinicia tu sistema:

    ```bash
    sudo reboot
    ```

    Esto es necesario antes de que los nuevos controladores CUDA 13.0 puedan ser instalados.
  </Step>

  <Step title="Completar la instalación">
    Después de que el sistema se reinicie, ejecuta el comando de instalación nuevamente:

    ```bash
    sudo zylon-cli install 1.5X.0
    ```

    Esta vez, la instalación se completará exitosamente con los nuevos controladores CUDA 13.0.
  </Step>
</Steps>
